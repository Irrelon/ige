Class IgePathComponent

Handles entity path traversal. This component is supposed to be added to individual entities wishing to traverse paths. When added to an entity the component will add a behaviour to the entity that is called each update() and will operate to move the entity along a defined path.

dynamic

  • dynamic(enable?): boolean | IgePathComponent

  • Gets / sets the dynamic mode enabled flag. If dynamic mode is enabled then at the end of every path point (reaching a tile along the path) the pathfinder will evaluate the path by looking ahead and seeing if the path has changed (the tiles along the path have now been marked as cannot path on). If any tile along the path up to the look-ahead value has been blocked, the path will auto re-calculate to avoid the new block.

    For dynamic mode to work you need to supply a pathfinder instance by calling .finder(), a tile checker method by calling .tileChecker() and the number of look-ahead steps by calling .lookAheadSteps(). See the doc for those methods for usage and required arguments.

    Parameters

    • Optional enable: boolean

      If set to true, enables dynamic mode.

    Returns boolean | IgePathComponent

    • Defined in src/engine/components/IgePathComponent.ts:124

overwrite

  • overwrite(eventName, id, listener): this

  • Overwrites any previous event listeners so that if the event fires, only the listener you pass will be called. If you pass an id along with the listener, only listeners for the id will be overwritten. This is similar to calling .off(eventName) then on(eventName, listener) since off(eventName) will cancel any previous event listeners for the passed event name.

    Parameters

    • eventName: string

      The name of the event to listen for.

    • id: string

      The id to match against.

    • listener: IgeEventListenerCallback

      The method to call when the event is fired.
